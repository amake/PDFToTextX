iPodBackup Help

Configuring an iPod

Before you can begin backing up, you must configure an iPod. This "registers" the iPod with iPodBackup so that the program knows what to do with it.

When iPodBackup launches, it checks to see if there are any mounted iPods. If there are mounted iPods, but no configured iPods, the Add iPod sheet will automatically appear.

To open the sheet manually, go to "Preferences..." in the iPodBackup application menu. Then click the button near the bottom of the window.

If you have more than one iPod mounted, choose the one you want to backup to from the popup menu. Be sure to check the iPod's free space before continuing! You can configure any number of iPods to be backed up with iPodBackup, each with its own backup style and rules.

You may choose to backup to either a folder or a disk image on the iPod. Both approaches have pros and cons.

Folder backups

When you configure a Folder backup, a new folder will be created at Your iPod/Users/username. The contents of your home folder will be copied into this folder.

Pros:

  • Folders are simple
  • Folders work well with booting OS X from your iPod

Cons:

  • Folders don't offer privacy
  • Manipulating folders with lots of items in them can be time-consuming
  • Folders can't be used on FAT32 (Windows) formatted iPods

Disk image backups

When you configure a Disk image backup, a blank disk image file will be created at Your iPod/Users/username.sparseimage. A sparse disk image is a special format of disk image that can dynamically grow in size as files are added to it. iPodBackup makes sparseimages with a maximum size of 95% the size of your iPod's hard drive.

The disk image can be either encrypted or unencrypted. The encryption is handled transparently by OS X.

Pros:

  • Disk images can be encrypted so your data is protected
  • Disk images work on FAT32 formatted iPods
  • A disk image is a single file, which can be easier to manipulate

Cons:

  • Disk images have to be mounted before you can access their contents
  • If you lose the password to an encrypted disk image, your data is likely gone forever. It is recommended that you have OS X store your disk image's password in the Keychain.
  • Similarly, if the disk image becomes corrupted, your data is likely gone forever. Don't try to view the disk image in iPodLinux's file browser!
  • Disk images can't be mounted on Windows machines
  • When deleting items from a disk image, the size of the disk image does not decrease unless you compact it. Compacting can be time consuming depending on the size of the image, and whether or not it's encrypted.

That said, the author of iPodBackup personally prefers disk images.
