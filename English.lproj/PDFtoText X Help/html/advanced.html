PDFtoText X Help

Advanced settings

PDFtoText X is a graphical frontend for a command line tool called pdftotext. Most of the advanced settings offered by PDFtoText correspond to the various arguments you can supply when invoking pdftotext in the Terminal.

Advanced settings are offered on the main window and in the Preferences window.

Main window

Descriptions below taken from the pdftotext manpage.

Setting Description Command line equivalent
Mimic original layout Maintain (as best as possible) the original physical layout of the text. The default is to 'undo' physical layout (columns, hyphenation, etc.) and output the text in reading order. -layout
Insert pagebreaks Don't insert page breaks (form feed characters) between pages (when unchecked). -nopgbrk (when unchecked)
Raw output Keep the text in content stream order. This is a hack which often "undoes" column formatting, etc. Use of raw mode is no longer recommended. -raw
Pages Specifies the first page to convert. -f
Specifies the last page to convert. -t
Line endings Sets the end-of-line convention to use for text output. (Default: unix) -eol
Encoding Sets the encoding to use for text output. (Default: UTF-8) -enc
Use password: Owner Specify the owner password for the PDF file. Providing this will bypass all security restrictions. -opw
Use password: User Specify the user password for the PDF file. -upw

Use the "Reset all" button to restore all settings to their defaults.

Preferences window

"Use custom pdftotext binary" allows you to specify a particular version of pdftotext to use in place of the one bundled with PDFtoText X. Note that unpredictable behavior may occur if the binary you specify is not compatible with the same command line options as the bundled version.
